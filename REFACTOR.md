# DouyinClaw 项目重构计划

## 问题分析

当前项目的主要问题：
1. **过度工程化** - 5 个 packages 的 monorepo 对于一个简单的续火花工具来说太重了
2. **选择器依赖** - Playwright + CSS 选择器在抖音改版时会失效
3. **部署复杂** - 需要 Redis + Docker + 多个服务

## 新的解决方案

### 核心思路
使用 **视觉 AI (GLM-4V)** 来理解页面，而不是依赖脆弱的 CSS 选择器。

### 架构对比

| 原方案 | 新方案 |
|--------|--------|
| Playwright + 选择器 | 视觉 AI + 简单浏览器控制 |
| 5 个 packages | 1-2 个核心模块 |
| Docker Compose | 单脚本运行 |
| 复杂部署 | 开箱即用 |

## 新的使用方式

```bash
# 1. 配置 GLM API Key
echo "GLM_API_KEY=your-key" >> .env

# 2. 运行续火花
pnpm spark:visual

# 3. (可选) 设置定时任务
pnpm spark:auto
```

## 保留的功能

1. **视觉浏览器代理** - 核心，可以复用到其他网站
2. **Web 管理后台** - 监控和配置
3. **AI 智能回复** - 视频分享回复
4. **飞书通知** - 状态推送
