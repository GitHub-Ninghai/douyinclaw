# 火花保持器 (Spark Keeper)

## 描述
自动为抖音好友续火花，保持社交活跃度。

## 触发条件
- 关键词: ["续火花", "火花", "发消息给好友"]
- 意图: social_maintenance
- 定时: 每日 9:00, 12:00, 18:00

## 依赖工具
- `get_friends`: 获取好友列表
- `check_spark`: 检测火花状态
- `screenshot`: 页面截图
- `send_message`: 发送消息

## 工作流程

### Step 1: 初始化
```
动作: 打开抖音私信面板
工具: browser_control(action="navigate", params={"url": "https://www.douyin.com"})
等待: 2秒
```

### Step 2: 获取好友列表
```
动作: 截图分析页面，识别好友列表
工具: screenshot()
工具: vision_analyze(任务="识别好友列表和火花图标")
输出: friends_list[]
```

### Step 3: 筛选需要续火花的好友
```
动作: 遍历好友列表，识别灰色火花图标
逻辑: 灰色 = 需要续火花, 彩色/橙色 = 已续，跳过
输出: need_spark_friends[]
```

### Step 4: 生成个性化消息
```
动作: 为每个需要续火花的好友生成消息
工具: chat_generate(类型="续火花", 好友信息)
规则:
  - 消息要有趣、个性化
  - 避免使用相同消息
  - 适当使用 emoji
  - 长度: 10-50 字
```

### Step 5: 发送消息
```
动作: 依次发送消息
工具: send_message(friend_id, message)
等待: 每次发送后等待 2-5 秒 (随机)
```

### Step 6: 记录结果
```
动作: 记录发送结果
输出:
  - 成功数量
  - 失败列表
  - 发送详情
```

## 错误处理

### 登录失效
```
检测: 页面显示登录按钮
处理: 通知用户重新登录
```

### 发送失败
```
检测: send_message 返回失败
处理:
  1. 重试一次
  2. 仍失败则跳过，记录错误
```

### 网络超时
```
检测: 操作超时
处理:
  1. 等待 5 秒
  2. 重新尝试
  3. 超过 3 次失败则停止
```

## 消息模板

### 早安型
- "早呀！来续火花啦~ 今天天气真好！☀️"
- "早安！新的一天，新的火花~ 🌅"
- "起床啦！该续火花了~ 🌞"

### 午间型
- "中午好！来续个火花~ 🍜"
- "午休时间，续个火花吧~ 😊"

### 晚间型
- "晚上好！今日份火花来啦~ 🌙"
- "睡前续个火花~ 晚安！💤"

### 节日型
- (根据节日自动调整)
- 周末: "周末愉快！来续火花~ 🎉"
- 节日: "{节日名}快乐！续火花啦~ 🎊"

## 注意事项

1. **频率控制**
   - 每个好友每天只发一条
   - 发送间隔 2-5 秒 (随机)
   - 避免被检测为机器人

2. **消息去重**
   - 记录已发送的消息
   - 连续 3 天不使用相同消息

3. **好友优先级**
   - 可以设置优先级列表
   - 优先处理高优先级好友

4. **静默模式**
   - 可配置静默时段 (如深夜)
   - 静默时段不发送消息

## 配置选项

```yaml
spark_keeper:
  enabled: true
  schedule:
    - "0 9 * * *"   # 每天 9:00
    - "0 12 * * *"  # 每天 12:00
    - "0 18 * * *"  # 每天 18:00

  message_style: friendly  # friendly | casual | formal
  max_friends_per_run: 20  # 每次最多处理好友数
  send_delay_min: 2        # 发送间隔最小值 (秒)
  send_delay_max: 5        # 发送间隔最大值 (秒)

  priority_friends:        # 优先处理的好友
    - "好友A"
    - "好友B"

  silent_hours:            # 静默时段
    start: "23:00"
    end: "07:00"
```
